莫队学习笔记

分类：
标签：

日期：2023-03-25 14:30:46

简介

莫队，用于处理区间问题的一种思想，本质是暴力，但是通过可以解决很多线段树等数据结构所解决不了的区间问题。

详解

例如对于小B的询问。
题意：

  • 小B有一个长为 nnn 的整数序列 aaa，值域为 [1,k][1,k][1,k]。
    他一共有 mmm 个询问，每个询问给定一个区间 [l,r][l,r][l,r]，求： ∑i=1kci2\sum\limits_{i=1}^k c_i^2i=1∑k​ci2​。
    其中 cic_ici​ 表示数字 iii 在 [l,r][l,r][l,r] 中的出现次数。
    小B请你帮助他回答询问。

分析

对于这道题，我们发现它有几个性质：

  1. 没有修改
  2. 区间之间的统计信息关系不确定
  3. 数据范围较小，但可以卡掉 O(n2)O(n^2)O(n2)
  4. 对于一个已知答案和 ccc 数组的序列，我们可以 O(1)O(1)O(1) 地增加或减少一个元素并维护其答案和 ccc 数组。
  • 对于性质1，我们可以离线。
  • 对于性质2，我们无法用线段树，前缀，st表等结构来维护，例如线段树的统计信息无法维护。
  • 对于性质4，具体这样实现的。比如我们增加一个元素 vvv 进来，相当于让答案从 cv2+…c_v^2+\dotscv2​+… 变成了(cv+1)2+…(c_v+1)^2+\dots(cv​+1)2+…，那么相当于答案增加 2×cv+12 \times c_v+12×cv​+1，cvc_vcv​ 加 111，减少元素也类似。

其实这时候已经很明显了，我们对于第一个查询暴力算出来答案和 ccc 数组，然后增加或减少若干元素并维护信息，求出第二个查询，然后在增加减少若干个元素并维护信息，求出第三个查询 …\dots…。那么我们就得到了一个暴力算法。

代码：

#include<iostream>
#include<vector>
using namespace std;
const int maxn=50010;
int n,m,m_,k,d,c[maxn],a[maxn];
long long ans,res[maxn];
vector<pair<int,int>>q;
int main(){
	ios::sync_with_stdio(false);
	cin.tie(0);cout.tie(0);
	cin>>n>>m>>k;
	for(int i=1;i<=n;i++)cin>>a[i];
	for(int i=1;i<=m;i++){
		pair<int,int>t;
		cin>>t.first>>t.second;
		q.push_back(t);
	}
	for(int i=q[0].first;i<=q[0].second;i++)c[a[i]]++;
	for(int i=1;i<=k;i++)ans+=c[i]*c[i];
	res[0]=ans;//暴力算第一个
    int l=q[0].first,r=q[0].second;
	for(int i=1;i<m;i++){
        while(l>q[i].first)ans+=(c[a[--l]]++)*2,ans++;
        while(r<q[i].second)ans+=(c[a[++r]]++)*2,ans++;
        while(l<q[i].first)ans-=(c[a[l++]]--)*2,ans++;
        while(r>q[i].second)ans-=(c[a[r--]]--)*2,ans++;//调整
		res[i]=ans;
	}
	for(int i=0;i<m;i++)cout<<res[i]<<endl;
  	return 0;
}

但是我们交上去，发现全部TLE了，那是因为 lll 和 rrr 可能会移动很大的范围，导致复杂度上升到 n2n^2n2。

那么我们是否可以限制他们移动的范围呢？没错，我们将所有询问离线下来，采用分块的思想，将 nnn 分为 [1,d],[d+1,2d]…[pd+1,n][1,d],[d+1,2d] \dots [pd+1,n][1,d],[d+1,2d]…[pd+1,n] 这些区域。

我们将 lll 在同一个区域的询问放在一起，然后对于同一个区域的询问按照 rrr 从小到大排序。这样的话，每个 lll 最多只会移动 ddd 次，所以 lll 一共最多会移动 mdmdmd 次，对于一个区域，rrr 由于是递增的，也最多只会移动 nnn 次，而一共有 n/dn/dn/d 个区域，所以 rrr 最多移动 n2/dn^2/dn2/d 次。 那么我们要移动的次数最小，就是要 md+n2/dmd+n^2/dmd+n2/d 最小了因此 ddd 取 n2/m\sqrt{n^2/m}n2/m​ 时时间复杂度最小，大约为 O(n×n)O(n \times \sqrt{n})O(n×n​)。

#include<algorithm>
#include<iostream>
#include<vector>
#include<cmath>
#include<map>
using namespace std;
const int maxn=50010;
int n,m,m_,k,d,c[maxn],a[maxn];
long long ans,res[maxn];
struct E{
    int l,r,id;
};
vector<E>q;
bool cmp(E x,E y){
    if(x.l/d!=y.l/d)return x.l/d<y.l/d;
    return x.r<y.r;
}
int main(){
    ios::sync_with_stdio(false);
    cin.tie(0);cout.tie(0);
    cin>>n>>m>>k;
    d=sqrt(n*n/m);
    for(int i=1;i<=n;i++)cin>>a[i];
    for(int i=1;i<=m;i++){
        E t;
        cin>>t.l>>t.r;
        t.id=i;
        q.push_back(t);
    }
    sort(q.begin(),q.end(),cmp);
    for(int i=q[0].l;i<=q[0].r;i++)c[a[i]]++;
    for(int i=1;i<=k;i++)ans+=c[i]*c[i];
    res[q[0].id]=ans;//暴力算第一个
    int l=q[0].l,r=q[0].r;
    for(int i=1;i<m;i++){
        while(l>q[i].l)ans+=(c[a[--l]]++)*2,ans++;
        while(r<q[i].r)ans+=(c[a[++r]]++)*2,ans++;
        while(l<q[i].l)ans-=(c[a[l++]]--)*2,ans++;
        while(r>q[i].r)ans-=(c[a[r--]]--)*2,ans++;
        res[q[i].id]=ans;
    }
    for(int i=1;i<=m;i++)cout<<res[i]<<endl;
    return 0;
}

树上莫队

树上莫队很简单，就是将树通过 dfsdfsdfs 序或者欧拉序转换为序列问题。

例题

例题Tree and Queries

题意

  • 给定一棵 nnn 个节点的树，根节点为 111。每个节点上有一个颜色 cic_ici​。mmm 次操作。操作有一种：
    1. u k：询问在以 uuu 为根的子树中，出现次数 ≥k\ge k≥k 的颜色有多少种。
  • 2≤n≤1052\le n\le 10^52≤n≤105，1≤m≤1051\le m\le 10^51≤m≤105，1≤ci,k≤1051\le c_i,k\le 10^51≤ci​,k≤105。

题解

我们直接对这个数做 dfsdfsdfs 序，那么询问 uuu 就相当于询问 [dfnu,dfnu+sizeu−1[dfn_u,dfn_u+size_u-1[dfnu​,dfnu​+sizeu​−1 （dfnu,sizeudfn_u,size_udfnu​,sizeu​ 分别表示 uuu 的 dfsdfsdfs 序和子树大小）

#include<algorithm>
#include<iostream>
#include<vector>
#include<cmath>
using namespace std;
const int maxn=500010;
int n,m,d,l,r,tot,siz[maxn],res[maxn];
int v[maxn],s[maxn];
vector<int>g[maxn];
struct E{
    int l,r,k,id;
}ask[maxn];
struct Node{
    int col,dfn,last;
}a[maxn];
void dfs(int x){
    a[x].dfn=++tot;
    siz[x]=1;
    for(int y:g[x]){
        if(!a[y].dfn)dfs(y),siz[x]+=siz[y];
    }
    a[x].last=tot;
    return;
}
bool cmp(E x,E y){
    if(x.l/d!=y.l/d)return x.l/d<y.l/d;
    return x.r<y.r;
} 
bool comp(Node x,Node y){
    return x.dfn<y.dfn;
}
int main(){
    ios::sync_with_stdio(false);
    cin.tie(0);cout.tie(0);
    cin>>n>>m;
    d=max((int)sqrt(n*n/m),1);
    for(int i=1;i<=n;i++)cin>>a[i].col;
    for(int i=1;i<n;i++){
        int u,v;cin>>u>>v;
        g[u].push_back(v);
        g[v].push_back(u);
    }
    dfs(1);
    for(int i=1;i<=m;i++){
        int x,k;cin>>x>>k;
        ask[i].id=i;
        ask[i].l=a[x].dfn;
        ask[i].k=k;
        ask[i].r=a[x].last;
    }
    sort(ask+1,ask+m+1,cmp);
    sort(a+1,a+n+1,comp);
    l=1;r=0;
    for(int i=1;i<=m;i++){
        while(l>ask[i].l)s[++v[a[--l].col]]++;
        while(r<ask[i].r)s[++v[a[++r].col]]++;
        while(l<ask[i].l)s[v[a[l++].col]--]--;
        while(r>ask[i].r)s[v[a[r--].col]--]--;
        res[ask[i].id]=s[ask[i].k];
    }
    for(int i=1;i<=m;i++)cout<<res[i]<<'\n';
    return 0;
}
